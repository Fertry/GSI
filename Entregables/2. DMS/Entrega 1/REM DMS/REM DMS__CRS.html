1 Introducción

Este proyecto se realiza para la asignatura de Gestión de Sistemas de Información durante el curso académico 2021/2022.

El proyecto consistirá en la realización de un sistema de gestión documental (o DMS) mediante la herramienta de Alfresco, para la empresa Salute Media, dedicada a la investigación de las enfermedades, y al desarrollo de las herramientas software necesarias para su tratamiento.

3 Descripción del sistema actual

El sistema actual es inexistente en tanto que la empresa requiere de un sistema de gestión documental del cual carece en este momento, y tan solo provee una serie de requisitos abajo detallados para la puesta en marcha de este.

Versión 1.0 (09/11/2021)

5.2 Casos de Uso

UC-0002 Creación de directorios
Versión 1.0 (09/11/2021)
Autores
  • Álvaro Chamorro Pascua
Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un usuario quiere crear una nueva carpeta dentro de un directorio en el que cuenta con permiso de escritura o durante la realización de los siguientes casos de uso: [UC-0006] Creación nuevo proyecto, [UC-0007] Registro nueva investigación.

Precondición

El usuario cuenta con permiso de escritura para el directorio dónde quiere operar.
Secuencia normal Paso Acción
1

El sistema permite el acceso al usuario a un directorio sobre el cual posee permiso de escritura.

2

El sistema permite al usuario definir una nueva carpeta dentro del directorio actual.
3

El sistema genera un nuevo recurso accesible para los actores con permiso (al menos) de lectura.
Postcondición

El nuevo contenido creado está a disposición de todos los usuarios con permiso (al menos de lectura) para ese directorio.
Importancia importante
Comentarios

Ninguno

UC-0004 Búsqueda de proyectos por metadatos
Versión 1.0 (09/11/2021)
Autores
  • Alejandro Fernández Trigo
Descripción

El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el usuario quiere hacer una consulta al sistema que devuelva un proyecto.

Precondición

El sistema está listo para recibir peticiones de búsqueda.
Secuencia normal Paso Acción
1

El sistema permite al usuario introducir un autor y/o palabra clave para ejecutar la búsqueda.
2

El actor Administrador (ACT-0001) introduce los términos de la búsqueda, lanzando así una petición.

3

El sistema devuelve (de existir) los recursos solicitados por la peticición, para los cuales el usuario tiene, al menos, permiso de lectura.
Postcondición

El recurso devuelto (de existir), está disponible para el usuario (este tiene, al menos, permiso de lectura) asociado.

Importancia importante
